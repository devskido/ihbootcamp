Day 10 - Requirements

과제 개요

이번 수업에서 JavaScript의 연산자(이항 산술: +, -, *, /, %, 단항 산술: ++, --, 단항 부정: !, 복합 대입: +=, -=, *=, /=, 비교: >, <, >=, <=, ==, ===, 논리: &&, ||, !, 삼항 연산자: ?:, 연산자 우선순위)와 조건문(if, else, else if, switch)을 배웠습니다. 이번 과제는 HTML 없이 순수 JavaScript를 사용하여 브라우저 콘솔에서 실행되는 학생 시험 점수 관리기를 제작하는 것입니다.

학생이 입력한 시험 점수(0~100)를 받아 최종 점수를 계산하고, 등급(A~F)을 부여하며, 합격/불합격 여부를 판단하고, 등급별 메시지를 출력합니다. 기본 과제는 연산자와 조건문(if로 등급 부여, switch로 메시지 출력)을 활용하여 핵심 기능을 구현하며, 도전 과제는 입력 유효성 검사와 추가 점수 조정을 추가합니다. 동일한 결과를 다양한 연산자 조합으로 구현할 수 있습니다.

기본 과제 요구사항

  1. JavaScript 파일 (score.js)
    • 브라우저 콘솔에서 실행 가능한 JavaScript 파일을 작성하세요.
    • 목표: 사용자가 입력한 시험 점수(0~100)를 받아 다음 정보를 출력:
      • 최종 점수: 입력 점수에 5점 보너스 추가(최대 105점).
      • 등급: 아래 범위에 따라 부여.
        • S: 100 이상
        • A: 90 이상 100 미만
        • B: 80 이상 90 미만
        • C: 70 이상 80 미만
        • D: 60 이상 70 미만
        • F: 60 미만
      • 합격/불합격 여부: 60점 이상이면 "Pass", 미만이면 "Fail".
      • 등급별 메시지: 아래 메시지를 정확히 출력.
        • S: "Super!!"
        • A: "Excellent work!"
        • B: "Good job!"
        • C: "Satisfactory performance."
        • D: "Needs improvement."
        • F: "Please try harder!"
    • 기능 요구사항:
      • 변수 선언:
        • var, let, const를 각각 최소 1회 사용하여 변수 선언(예: 점수, 최대 점수, 등급).
        • 예: const maxScore = 100; (최대 점수), let score; (입력 점수), var grade; (등급).
      • 연산자 사용:
        • 이항 산술: 보너스 점수 추가(예: score + 5).
        • 복합 대입: 점수에 5점 추가(예: score += 5 또는 score = score + 5).
        • 비교: 등급 범위 확인(예: score >= 90).
        • 논리: 등급 범위 조건(예: score >= 90 && score <= 100).
        • 삼항 연산자: 합격/불합격 판단(예: score >= 60 ? "Pass" : "Fail").
        • 연산자 우선순위: 계산 시 괄호 사용 권장(예: (score + 5) <= 100).
        • 학생은 연산자 조합을 자유롭게 선택 가능(예: score + 5 vs score += 5).
      • 조건문:
        • 등급 부여: if, else if, else만 사용하여 등급(A, B, C, D, F) 부여.
        • 메시지 출력: switch만 사용하여 등급별 메시지 출력(위 메시지 정확히 준수).
      • 입출력:
        • prompt()로 점수(0~100)를 입력받음.
        let input = prompt("점수를 입력하세요.");
console.log(input);
        • 콘솔 출력 형식:
        Final Score: [최종 점수]
Grade: [등급]
Status: [Pass/Fail]
Message: [등급별 메시지]
        • 예: 입력 85 → 출력:
        Final Score: 90
Grade: A
Status: Pass
Message: Excellent work!
    • 구현 선택:
      • 연산자 활용 방식은 자유롭게 조합(예: score = score + 5 vs score += 5).
      • if 문의 조건 순서와 논리 연산자는 학생이 설계 가능(예: score >= 90 vs score > 89).
      • switch 문에서 case는 등급(A, B, C, D, F)에 맞춰 작성.
  2. 실행 방법
    • JavaScript 파일을 작성 후, 브라우저 개발자 도구(F12 → Console)에서 코드를 복사/붙여넣기하여 실행하거나, 간단한 HTML 파일에 <script src="score.js"></script>를 추가하여 실행:
    • HTML은 제출에 포함하지 않아도 됩니다.
  3. 파일 구조
    • 프로젝트 폴더를 구성:
    score-calculator/
├── score.js

도전 과제 (선택)

기본 과제를 완료한 학생들은 추가 연산과 조건 처리를 연습하세요.

  1. 추가 기능 구현
    • 입력 유효성 검사:
      • 입력 점수가 0~100 범위를 벗어나거나 숫자가 아닌 경우(isNaN(score)), 정확히 Invalid score! Please enter a number between 0 and 100. 출력 후 프로그램 종료.
      • 논리 연산자(||, !) 필수(예: isNaN(score) || score < 0 || score > 100).
    • 추가 연산:
      • 단항 산술: 점수 조정(예: score++로 1점 증가 후 5점 보너스 추가).
      • 복합 대입: 점수 스케일링(예: score *= 1.1로 10% 가산).
      • 단항 부정: 조건 반전(예: !isNaN(score)).
    • 복잡한 조건:
      • 보너스 점수와 스케일링 후 점수가 100을 초과하면 100으로 제한(예: if (score > 100) score = 100;).
      • switch에 특수 case 추가: 점수가 정확히 100이면 메시지 Perfect Score! 출력(A 등급 메시지 대신).
    • 출력 형식:
      • 유효하지 않은 입력:
      Invalid score! Please enter a number between 0 and 100.
      • 유효한 입력(예: 입력 90):
      Final Score: 100
Grade: A
Status: Pass
Message: Perfect Score!
    • 구현 선택:
      • 유효성 검사는 if 또는 삼항 연산자로 가능.
      • 점수 조정은 ++, =, += 등 조합 자유.
      • switch 문에서 score === 100을 별도 case로 처리하거나 A 등급 내에서 조건 추가 가능.

주의사항

  • JavaScript 문법: var, let, const, 연산자(이항 산술, 단항 산술, 단항 부정, 복합 대입, 비교, 논리, 삼항), 조건문(if, else if, else로 등급 부여, switch로 메시지 출력)만 사용하세요. 배열, 객체, 루프, 함수, DOM 조작은 사용하지 마세요.
  • 실행 환경: 브라우저 콘솔(F12)에서 실행하거나, 간단한 HTML 파일로 테스트하세요. Node.js는 사용하지 않습니다.
  • 코드 가독성: JavaScript 코드는 들여쓰기를 통해 가독성을 높이고, 주석(//)으로 주요 코드(연산자 사용, 등급 부여, 메시지 출력)를 설명하세요.
  • 테스트: 브라우저 콘솔에서 score.js를 실행하여 최종 점수, 등급, 합격 여부, 메시지가 정확히 출력되는지 확인(예: 입력 85 → Final Score: 90, Grade: A, Status: Pass, Message: Excellent work!).
  • 입력 처리: prompt()는 문자열을 반환하므로 Number()로 변환하세요. 도전 과제에서는 유효성 검사 필수.
